Overview:
This was a project done for the class IMM 280: Design Perspectives in collaboration with Trentoniana. Trentoniana is a branch of the Trenton Public Library which houses artifacts from historical Trenton and other areas in New Jersey.

The Problem:

Create a piece of interactive media that visitors of Trentoniana can use to further their knowledge about Trenton in a more interesting manner than traditional reading and learning.

Research

We decided to focus on the Hall Mills Case from finding newspaper clippings while looking through Trentoniana.



Inspiration:
Our inspiration for the game aspect of the website itself was the Nancy Drew Online Mini-Games. Our inspiration for the mood and tone was the Buzzfeed Unsolved video series from Buzzfeed.

Second Iteration:
Wireframes

We changed the functionality of the game by making it multiple pages in the case file in which the user can ‚Äúflip‚Äù through the pages. The user can click back and forth between pages.

Why the redesign:
  • User feedback: The people from Trentoniana suggested having multiple pages as opposed to one static page to interact with, the team agreed with them.
  • Overall having multiple pages would create a more active experience as there are more options for the user to interact with the case file itself. Along with this, it would also lessen the short-term memory load on the user as there will be a little information on several pages as opposed to a lot of information.

New Problem:
How do we simulate that the user is flipping through pages in the case file?

Solution:
Have different colored tabs, start with them all on one side of the case file, have them switch to the other side when moving to the next page to simulate that some pages have passed and there are more pages to go.




We then had our professor do a cognitive walkthrough of the game as our user testing for this iteration of the design process.

Key Feedback
  • Move the page turn links above
  • Eliminate scrolling
  • Just have previous and next as opposed to having ‚ÄúPrevious - Title of Page‚Äù and ‚ÄúNext - Title of Page‚Äù
  • Mathematically correct (mixing navigation with information)
Final Changes
  • We changed navigation links from a long description to just "previous" and "next"
  • Why the redesign:
    User feedback, we agreed with our professor to change the navigation links both in content and location. It is easier for the user as there is less to read while still accomplishing the same task as before, we simplified the step.

Solution:

The Pitch:
What if you could solve one of the oldest unsolved crimes in history?

The Hall-Mills murder is one of the nation's oldest unsolved crimes, taking place in New Jersey in 1922. In this web-application based game users will be able to click through an interactive case file to look through clues, hear testimonials from witnesses, and finally to "vote" on who they think committed the crime.


1st page/cover page in the case file

2nd page in the case file, basic summary of case

Symbols Page

On this page the user clicks on each symbol in order to learn the significance of it within the case. Once a symbol is clicked a popup appears with a clue.

Suspects Page

On this page the user clicks on each suspect in order to hear their testimonials, alibis and anything else that is relevant to the case. Once a suspect is clicked a popup with a video appears with the suspects explaining their sides of the story.



Last Page

From Here users can scroll down and "solve" the case by voting on who they think comitted the crime.

Looking Back: What I Would Change

Overall I would advocate for more user testing throughout the only people we had tested out the project were the clients and the professor. While it was important to please both the professor and the client, they aren't the target audience for the game.

What I Would Suggest To My Past Self:
  • Before designing - make sure the content is okay for kids, find a good age range, and see if it is a topic they find interesting.
  • During Development - have several kids test out the game and see if they know how to navigate and iterate the design based on what they said, have other college students test it out to see if it is a project they find interesting.
  • Final Testing - have several kids test out the game and see if they know how to navigate and iterate the design based on what they said, have other college students test it out to see if it is a project they would find interesting.
What I Learned

This project was my first introduction to UX and the design process, I learned a lot about collaboration and iterating ideas based on user feedback. This is the project where I decided to pursue UI/UX so I am very grateful for it.
